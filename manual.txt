서버 여는 법과 닫는 법

1. 터미널 열기

PowerShell 또는 명령 프롬프트를 엽니다.

2. 프로젝트 폴더로 이동하기

아래 명령어를 입력합니다.

cd "D:\학습공동체"

3. 서버 열기

아래 명령어를 입력합니다.

$env:OPENAI_API_KEY="<OpenAI_API_KEY>"
python server.py 8000

정상적으로 실행되면 터미널에 서버가 실행 중이라는 문구가 표시됩니다.
이 프로젝트는 회원가입/로그인 기능과 캐시 방지 설정이 들어간 server.py를 사용합니다.

OPENAI_MODEL을 따로 지정하지 않으면 app.js에서 전달한 기본 모델을 사용합니다.
서버에서 모델을 강제로 지정하고 싶으면 아래처럼 실행합니다.

PowerShell:

$env:OPENAI_API_KEY="<OpenAI_API_KEY>"
$env:OPENAI_MODEL="gpt-5.4-mini"
python server.py 8000

브라우저에서 아래 주소로 접속합니다.

http://localhost:8000

특정 파일을 바로 보고 싶으면 아래처럼 접속할 수 있습니다.

http://localhost:8000/index.html

4. 서버 닫기

서버를 실행한 터미널 창에서 아래 키를 누릅니다.

Ctrl + C

종료할지 물어보면 Y를 입력하고 Enter를 누릅니다.

터미널 창을 닫았거나 서버가 백그라운드에서 실행 중이면 아래 명령어로 8000번 포트를 사용하는 프로세스를 확인합니다.

Get-NetTCPConnection -LocalPort 8000 -State Listen | Select-Object LocalAddress,LocalPort,OwningProcess

OwningProcess에 표시된 숫자가 17192라면 아래처럼 종료합니다.

Stop-Process -Id 17192

5. 같은 와이파이에 연결된 다른 기기에서 접속하기

같은 와이파이에 연결된 휴대폰, 태블릿, 다른 컴퓨터에서는 localhost 주소를 사용할 수 없습니다.
localhost는 서버를 연 PC 자기 자신을 뜻합니다.

다른 기기에서는 서버를 연 PC의 IP 주소로 접속해야 합니다.
현재 이 PC의 접속 주소는 아래와 같습니다.

http://192.168.0.2:8000

최신 수정이 바로 안 보이면 아래처럼 주소 끝에 버전값을 붙여 접속합니다.

http://192.168.0.2:8000/index.html?v=latest

예를 들어 서버를 열어 둔 상태에서 휴대폰 브라우저에 위 주소를 입력하면 접속할 수 있습니다.

만약 접속이 안 되면 아래 내용을 확인합니다.

- 서버를 연 PC와 접속하려는 기기가 같은 와이파이에 연결되어 있는지 확인합니다.
- Windows 방화벽에서 Python 또는 해당 서버 프로그램의 개인 네트워크 접근을 허용합니다.
- 서버를 열 때 사용한 포트 번호가 8000번인지 확인합니다.
- 예전 화면이 계속 보이면 새 탭에서 열거나 주소 끝에 ?v=latest 같은 값을 붙입니다.

6. 내 IP 주소 다시 확인하기

IP 주소는 네트워크 환경에 따라 바뀔 수 있습니다.
다시 확인하려면 터미널에서 아래 명령어를 입력합니다.

ipconfig

그 다음 인터넷에 연결된 어댑터의 IPv4 주소를 확인합니다.
현재는 아래 주소를 사용합니다.

192.168.0.2

7. 8000번 포트가 이미 사용 중일 때

아래처럼 다른 포트 번호로 실행합니다.

python server.py 8080

그 다음 브라우저에서 아래 주소로 접속합니다.

http://localhost:8080

같은 와이파이의 다른 기기에서는 아래 주소로 접속합니다.

http://192.168.0.2:8080

8. Python 명령어가 안 될 때

아래 명령어를 대신 사용해 봅니다.

py server.py 8000

그래도 실행되지 않으면 Python이 설치되어 있지 않은 상태일 수 있습니다.

단순 정적 파일 확인만 필요하면 아래 명령어도 사용할 수 있습니다.

py -m http.server 8000

하지만 이 경우 회원가입/로그인 API와 server.py의 캐시 방지 설정은 동작하지 않을 수 있습니다.

1. 프로젝트 폴더로 이동

cd "D:\학습공동체"

2. 최신 코드 받기

git pull

Already up to date.가 나오면 현재 컴퓨터는 원격 저장소의 최신 상태입니다.

python server.py 8000

4. 브라우저에서 캐시 방지 주소로 접속

내 컴퓨터:

http://localhost:8000/index.html?v=latest

같은 와이파이의 다른 기기:

http://192.168.0.2:8000/index.html?v=latest

5. 그래도 이전 화면이면 브라우저 탭을 닫고 새 탭에서 다시 엽니다.

6. 검색창, AI 상담 버튼, 레인보우 버튼 같은 JavaScript/CSS 수정이 안 보이면 HTML에서 app.js와 styles.css 링크에 버전값이 붙어 있는지 확인합니다.

예시:

styles.css?v=ai-20260525-1
app.js?v=ai-20260525-1

7. GitHub 커밋/푸시가 API 키 때문에 막힐 때

app.js, server.py, manual.txt 같은 파일에 실제 API 키가 들어갔는지 확인합니다.

아래 명령어로 실제 키 패턴을 검색합니다.

rg "sk-[A-Za-z0-9_-]{20,}|sk-proj-|apiKey:"

검색 결과에 실제 sk-로 시작하는 긴 키가 나오면 그 키를 코드에서 삭제합니다.
이 프로젝트의 AI 상담은 /api/chat 서버 API를 통해 동작하며, server.py가 OPENAI_API_KEY 환경변수를 읽어 OpenAI에 요청합니다.

.env 파일이나 data 폴더는 Git에 올리지 않습니다.
이미 실제 API 키를 GitHub에 올리려고 했거나 코드에 저장한 적이 있으면 OpenAI 대시보드에서 기존 키를 폐기하고 새 키를 발급합니다.

8. API 키 관련 현재 구조

app.js:

- OpenAI API 키를 저장하지 않습니다.
- 사용자가 AI 상담에 질문하면 /api/chat으로 요청합니다.
- 브라우저에는 API 키가 절대 내려가면 안 됩니다.

server.py:

- /api/chat 요청을 받습니다.
- OPENAI_API_KEY 환경변수를 읽습니다.
- 서버에서만 OpenAI API로 요청을 보냅니다.

.gitignore:

- .env
- .env.*
- data/
- __pycache__/
- *.pyc

위 파일과 폴더는 Git에 올리지 않습니다.

9. API 키 수정 후 커밋하기

아래 순서로 확인한 뒤 커밋합니다.

python -m py_compile server.py
rg "sk-[A-Za-z0-9_-]{20,}|sk-proj-|apiKey:"
git status

rg 검색 결과에 실제 API 키가 나오지 않으면 커밋합니다.

git add .
git commit -m "Move OpenAI API key to server environment"
git push

10. Render 같은 배포 서버에서 API 키 설정하기

배포 서버에서는 코드에 키를 넣지 않고, 서비스 설정의 Environment Variables에 아래 값을 추가합니다.

OPENAI_API_KEY=<OpenAI_API_KEY>

필요하면 모델도 환경변수로 추가합니다.

OPENAI_MODEL=gpt-5.4-mini

환경변수를 추가하거나 수정한 뒤에는 서버를 재배포하거나 재시작합니다.
